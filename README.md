# Varangians LMS

Status: not yet started — project name, one-line description and scope are being established during discovery.

A fresh .NET implementation of the Varangians youth-programme learning/progression platform. A prior Power Platform-based build of the same project was abandoned; this repository starts clean, carrying forward only the underlying business requirements, not its tooling or design decisions.

See `docs/` for the working requirements and design documentation (the source of truth):

- `docs/requirements/` — problem statement, stakeholders, user journeys, business/functional/non-functional requirements, constraints, acceptance criteria
- `docs/design/` — architecture (HLD), low-level design, data design, integration
- `docs/quality/` — test plan, traceability matrix
- `docs/governance/` — RAID log, security & compliance
- `docs/adr/` — architecture decision records
- `docs/delivery-plan.md`, `docs/glossary.md` — plan and shared terminology

Rendered gate packages (branded `.docx`, generated from the markdown above) live in `gates/design/` and `gates/delivery/` — never hand-authored.
